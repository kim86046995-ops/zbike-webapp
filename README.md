## 📊 데이터 아키텍처

### 저장소 (Storage)

#### Cloudflare D1 (SQLite) - 구조화된 데이터
텍스트 데이터, 관계형 데이터 저장 (최대 1MB/컬럼)

#### Cloudflare R2 Storage - 파일 저장소 🆕
- **신분증 이미지** 저장 (D1 1MB 제한 회피)
- **버킷**: `zbike-id-cards`
- **바인딩**: `R2_ID_CARDS`
- **파일 경로**: `id-cards/{timestamp}-{random}.jpg`
- **공개 URL**: R2 public domain을 통한 접근
- DB에는 R2 URL만 저장 (텍스트)

### 데이터베이스: Cloudflare D1 (SQLite)

### API 엔드포인트

#### 인증 API (Authentication)
- `POST /api/auth/login` - 로그인 (username, password)
- `POST /api/auth/logout` - 로그아웃 (세션 ID 필요)
- `GET /api/auth/check` - 세션 유효성 확인
- `POST /api/auth/find-username` - 아이디 찾기 (name, phone)
- `POST /api/auth/request-reset` - 비밀번호 재설정 토큰 생성 (username, name, phone)
- `POST /api/auth/reset-password` - 비밀번호 재설정 (token, new_password)

#### 오토바이 API (인증 필요)
- `GET /api/motorcycles` - 목록 조회 (상태 필터 가능)
- `GET /api/motorcycles/:id` - 상세 조회
- `GET /api/motorcycles/:id/contracts` - **오토바이별 계약 이력 조회**
- `POST /api/motorcycles` - 등록 ⚠️ **인증 필요**
- `PUT /api/motorcycles/:id` - 수정 ⚠️ **인증 필요**
- `DELETE /api/motorcycles/:id` - 삭제 ⚠️ **인증 필요**

#### 고객 API
- `GET /api/customers` - 목록 조회
- `GET /api/customers/:id` - 상세 조회
- `POST /api/customers` - 등록
- `PUT /api/customers/:id` - 수정

#### 계약서 API (인증 필요)
- `GET /api/contracts` - 목록 조회 (JOIN으로 관련 정보 포함)
- `GET /api/contracts/:id` - 상세 조회
- `POST /api/contracts` - 생성 ⚠️ **인증 필요**

#### 차용증 API 🆕

- `GET /api/loan-contracts/:id` - 상세 조회
- `POST /api/loan-contracts` - 생성
- `PATCH /api/loan-contracts/:id/status` - 상태 변경

#### 업체 관리 API 🆕
- `POST /api/upload/id-card` - 신분증 이미지 업로드 (R2 Storage)
  - Request: `{ image: "data:image/jpeg;base64,..." }`

- `GET /api/companies` - 업체 목록 조회
- `GET /api/companies/search?q=<query>` - 업체 검색

1. 로그인 페이지에서 "아이디 또는 비밀번호를 잊으셨나요?" 클릭
2. **아이디 찾기**
   - 이름과 전화번호 입력
   - 일치하는 아이디 표시
3. **비밀번호 찾기**
   - 1단계: 아이디, 이름, 전화번호로 본인 확인
   - 2단계: 6자리 인증번호 확인 (30분 유효)
   - 3단계: 새 비밀번호 설정
   - 완료 후 로그인 페이지로 이동

### 1. 오토바이 등록
1. 로그인 후 메인 페이지에서 **빠른 액세스 > "오토바이 등록"** 클릭
2. 전용 등록 페이지에서 모든 필수 정보 입력
   - 차량번호, 차량이름, 차대번호
   - 키로수, 연식
   - 보험 정보 (보험사, 시작일, 종료일, 운전범위)
   - 명의자, 보험료, 차량금액
   - 사용 현황 메모 (선택)
3. "등록하기" 버튼 클릭
4. 등록 완료 후 자동으로 오토바이 목록 페이지로 이동

### 1-1. 오토바이 관리 (조회/수정/삭제)
1. 메인 페이지에서 "총바이크: X대" 클릭
2. 오토바이 목록 화면에서:
   - 검색 (번호판/계약자/명의자)
   - 상태 필터링 (전체/사용가능/렌트중/정비중/폐지)
   - 정렬 (최신순/번호판순/보험만료순 등)
3. 각 오토바이 카드에서:
   - **번호판 클릭**: 계약 이력 조회
   - **수정 버튼**: 정보 수정
   - **삭제 버튼**: 오토바이 삭제
   - **해지 버튼**: 모든 진행중인 계약 해지

### 2. 보험 기간 확인
1. 오토바이 관리 페이지에서 각 오토바이 카드 확인
2. 보험 기간 프로그레스 바 색상 확인
   - **파란색**: 안전 (30일 이상 남음)
   - **주황색**: 경고 (30일 미만 남음)
   - **빨간색**: 위험 (만료됨)
3. 남은 일수 실시간 표시 확인

### 3. 계약 이력 조회
1. 오토바이 관리 페이지에서 **번호판 클릭**
2. 해당 오토바이의 전체 계약 이력 모달 표시
3. 계약 상태, 고객 정보, 계약 상세 확인

### 4. 계약서 작성
1. 로그인 후 메인 페이지에서 "계약서 작성" 클릭
2. 계약 타입 선택 (리스/렌트)
3. 사용 가능한 오토바이 선택
4. 고객 정보 입력
5. 계약 상세 정보 입력 (기간, 금액 등)
6. 서명 박스에 고객 서명 받기
7. "계약서 생성" 버튼 클릭

### 5. 계약서 조회
1. 로그인 후 메인 페이지에서 "계약서 목록" 클릭
2. 필터/검색으로 원하는 계약서 찾기
3. "상세보기" 버튼으로 전체 내용 확인

### 6. 로그아웃
1. 메인 페이지 우측 상단 "로그아웃" 버튼 클릭
2. 확인 후 로그인 페이지로 이동

## ⚙️ SMS 설정 가이드

### CoolSMS 가입 및 설정

1. **CoolSMS 가입**
   - https://www.coolsms.co.kr/ 접속
   - 회원가입 및 로그인

2. **발신번호 등록**
   - 대시보드 > 발신번호 관리
   - 사업자 등록증 또는 신분증 제출
   - 발신번호 승인 (1-2영업일)

3. **API Key 발급**
   - 대시보드 > API Key 관리
   - API Key 및 API Secret 생성
   - 테스트 크레딧 받기 (신규 가입 시 2,000원 제공)

4. **환경변수 설정**
   
   `.dev.vars` 파일에 다음 정보 입력:
   ```env
   # CoolSMS API 설정
   COOLSMS_API_KEY=your_api_key_here
   COOLSMS_API_SECRET=your_api_secret_here
   COOLSMS_SENDER=010-1234-5678
   ```

5. **프로덕션 배포 시 환경변수 설정**
   ```bash
   npx wrangler secret put COOLSMS_API_KEY
   npx wrangler secret put COOLSMS_API_SECRET
   npx wrangler secret put COOLSMS_SENDER
   ```

### SMS 요금 안내
- 일반 SMS (90바이트): 약 15원
- LMS (장문, 2000바이트): 약 45원
- 계약서 링크 전송 시 LMS 사용 권장

### SMS 전송 플로우
1. 관리자가 계약서 작성 완료
2. "고객에게 전송" 버튼 클릭
3. SMS 또는 카카오톡 선택
4. 고객 휴대폰 번호로 링크 전송
5. 고객이 링크 클릭하여 계약서 검토
6. 전자서명 및 신분증 업로드
7. 제출 시 자동으로 계약서 생성
