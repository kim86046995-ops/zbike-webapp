# 관리자 관리 기능 구현 완료

## 개요
슈퍼관리자가 관리자를 구분하고, 관리자 계정의 사용중지/허가를 할 수 있는 기능이 추가되었습니다.

## 구현 내용

### 1. 백엔드 API
**PATCH /api/admin/users/:username/status** - 관리자 상태 변경 (슈퍼관리자 전용)
- 슈퍼관리자만 접근 가능
- 자기 자신은 변경 불가
- 슈퍼관리자는 정지 불가
- 비활성화 시 모든 세션 자동 삭제
- 상태값: `active` (사용허가), `inactive` (사용중지)

**요청 예시:**
```bash
curl -X PATCH "http://localhost:3000/api/admin/users/testadmin/status" \
  -H "Content-Type: application/json" \
  -H "X-Session-ID: <session_id>" \
  -d '{"status":"inactive"}'
```

**응답 예시:**
```json
{
  "success": true,
  "message": "사용자가 비활성화되었습니다"
}
```

### 2. 프론트엔드 UI

#### 로그인 상태 표시
**헤더 우측 상단:**
- **슈퍼관리자**: `[관리자명] [슈퍼관리자 배지]` (보라색)
- **관리자**: `[관리자명] [관리자 배지]` (회색)
- 사용자 이름 클릭 시 관리자 관리 모달 열기 (슈퍼관리자만)

#### 관리자 관리 모달 (슈퍼관리자 전용)
**표시 조건:**
- 슈퍼관리자만 접근 가능
- 일반 관리자는 모달이 열리지 않음

**모달 구성:**
- 제목: "관리자 관리"
- 관리자 목록:
  - 이름, 아이디, 이메일
  - 역할 배지 (슈퍼관리자/관리자)
  - 상태 배지 (사용중/사용중지)
  - 상태 변경 버튼:
    - **사용중**: [사용중지] 버튼 (빨간색)
    - **사용중지**: [사용허가] 버튼 (초록색)

### 3. 사용자 플로우

#### 슈퍼관리자 플로우:
1. **로그인**: 대시보드 우측 상단에 "관리자 [슈퍼관리자]" 표시
2. **이름 클릭**: 관리자 관리 모달 열림
3. **관리자 목록 확인**: 모든 관리자 목록 표시
4. **상태 변경**:
   - 사용중지: 확인 다이얼로그 → 비활성화 → 세션 삭제
   - 사용허가: 확인 다이얼로그 → 활성화
5. **결과 확인**: 알림 메시지 표시 → 목록 자동 새로고침

#### 일반 관리자 플로우:
1. **로그인**: 대시보드 우측 상단에 "관리자 [관리자]" 표시
2. **이름 클릭**: 아무 동작 없음 (슈퍼관리자만 접근 가능)
3. **상태 정지 시**: 로그인 불가 (세션 삭제됨)

### 4. 보안 정책

#### 슈퍼관리자 보호:
- 슈퍼관리자는 정지할 수 없음
- 자기 자신의 상태는 변경 불가

#### 세션 관리:
- 비활성화 시 모든 세션 자동 삭제
- 비활성화된 관리자는 로그인 불가

#### 권한 검증:
- 모든 API 호출 시 세션 검증
- 슈퍼관리자 권한 필수

### 5. UI 디자인

#### 역할 배지:
- **슈퍼관리자**: 보라색 그라데이션 (`bg-gradient-to-r from-purple-500 to-purple-600`)
- **관리자**: 회색 (`bg-gray-500`)

#### 상태 배지:
- **사용중**: 초록색 (`bg-green-500`)
- **사용중지**: 빨간색 (`bg-red-500`)

#### 버튼:
- **사용중지**: 빨간색 (`bg-red-500 hover:bg-red-600`)
- **사용허가**: 초록색 (`bg-green-500 hover:bg-green-600`)
